Current Limitations

This document describes the current limitations of SEJ as of release 0.3.2. In this early release, many things are still incomplete or missing entirely.

Spreadsheet Formats

  • Excel .xml support is not part of the test suite right now and, therefore, probably broken.
  • OpenOffice Calc spreadsheet loader is not done at all.
  • The Excel .xls loader does not yet handle:
    • references to other sheets
    • labels in formulas

Excel Functionality

  • SEJ currently supports only the basic numeric operators (+, -, *, /, %, ^), SUM, MIN, MAX, AVERAGE, PRODUCT, IF, ROUND. A good overview of the supported functionality can be gained from this file of test cases.
  • Range intersections are parsed but not evaluated correctly.
  • You cannot bind to rightmost or bottommost cells that have no value (because Excel does not save them).

Scaled Long Type

  • Currently computes exponentiation and conversion from and to dates using the double type.
  • Scaled division currently swaps arguments on the stack a little too much.

BigDecimal Type

  • Can only compute exponentiation with integer exponents. The exponent is silently truncated.

Interfacing with Java

  • Subsections of varying width or height, while they can be defined, are not handled by the compiler yet.
  • Only double-, BigDecimal-, Date-, and boolean-valued computations are supported. In particular, there is no support for String and null yet.
  • The compiler cannot handle parametrized outputs yet.

Engine Persistence

  • The engines stored by SEJ are not yet in the definitive format. It is currently just a single class file's contents, with minimal versioning information. This will change to a more robust format in the future.

Miscellaneous

  • The interactive demo is broken (because of the missing parametrized outputs).
