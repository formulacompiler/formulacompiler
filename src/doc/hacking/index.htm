Contributing To SEJ

Thank you! I appreciate that you want to contribute to SEJ. In these pages I will tell you how to go about it. We have to cover quite a bit of ground, as SEJ was and is developed with great attention to quality. This means there are a few rules to follow:

  1. Document first
  2. Cite tested examples
  3. Mimick existing code
  4. Implement only what the tests require

But never fear, the examples will make them clear.
