Release Notes

Release 0.3.1

This minor release changes the name Compiler.NameSpace to Compiler.Section. Since we are no longer binding by name, the term NameSpace is no longer appropriate.

In addition Compiler.getRootNameSpace() has been shortened to Compiler.getRoot().

Release 0.3.0

This is the first release with the new API based on binding cells to native types. The interpreter has been dropped and replaced by a byte code compiler producing very fast code.

Limitations

  • Excel .xml support is still limited.
  • OpenOffice Calc ist not supported at all yet.
  • SEJ currently supports only the basic numeric operators (+, -, *, /, %, ^), SUM, MIN, MAX, AVERAGE, PRODUCT, IF, ROUND. A good overview of the supported functionality can be gained from this file of test cases.
  • Subsections of varying width or height are not handled by the compiler yet. Support for defining them is already in place.
  • IF is only supported for arguments that are direct comparisons, not arbitrary boolean expressions.
  • You cannot bind to rightmost or bottommost cells that have no value (because Excel does not save them).
  • Only double- and Date- valued computations are supported. In particular, there is no support for null.
  • The compiler cannot handle parametrized outputs yet.
  • The interactive demo is broken (because of the missing parametrized outputs).
